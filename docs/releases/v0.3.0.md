# OpenGrove v0.3.0

OpenGrove v0.3.0 turns the workspace into an App-first operating surface: mounted Apps can expose files, chat, previews, developer sessions, and visual annotations without forcing every workflow through the original chat-only shell.

## Highlights

- Added the mounted App workspace: App navigation, file workbench, App-scoped chat, App package metadata, App creation/import flows, and reusable directory/file preview components.
- Added App developer mode with persistent developer sessions, visual preview workbench, element/box/stroke/note annotations, annotation context packets, and preview reload/status handling.
- Added extension inventory and managed skill publishing for mounted skills, CLIs, MCP config, hooks, plugins, and tool roots.
- Expanded settings for kernels, providers, mounted Apps, voice input, remote Matrix/Tuwunel messaging, proxy, appearance, diagnostics, and GitHub Copilot CLI login.
- Reworked Rooms and Contacts around explicit contact entities and local Room Ledger state, with optional Matrix/Tuwunel projection stored as generic remote provenance.
- Refined the shell UI: sidebar overflow, settings surfaces, contacts, room/member management, mounted App chat composer chrome, and developer annotation toolbar cleanup.
- Refreshed public documentation, including App spec, core decisions, technical reference, design guide, release process, and bilingual docs.

## Configuration Notes

- Restart the OpenGrove local bridge after upgrading so new routes such as `/developer/sessions`, App routes, extension routes, and voice routes are loaded by the running process.
- Mounted Apps should declare their package boundary in `opengrove.app.json`; `ui.kind` can point to a file workbench or a web UI preview.
- Matrix/Tuwunel remains optional. Local Rooms own the primary state; remote room identifiers and sync metadata live under generic `remote` provenance.
- Voice input remains available in the normal chat composer. The unused visual-developer toolbar voice action was removed from the annotation tools.

## Verification

Passed before release:

- `npm run check`
- `npm run test:harness`
- `npm run build:web`
- `npm pack --dry-run`

# OpenGrove v0.3.0 中文说明

OpenGrove v0.3.0 把工作区推进到以 App 为中心的操作界面：已加载 App 可以暴露文件、聊天、预览、开发者会话和视觉标注，不再把所有工作流都压回原来的单一聊天壳。

## 亮点

- 新增已加载 App 工作区：App 导航、文件工作台、App 作用域聊天、App 包元数据、App 创建/导入流程，以及可复用的目录树和文件预览组件。
- 新增 App 开发者模式：持久化 developer sessions、视觉预览工作台、元素/区域/画线/备注标注、annotation context packets，以及预览刷新和状态处理。
- 新增扩展 inventory 和托管 skill 发布流程，覆盖 mounted skills、CLI、MCP config、hooks、plugins 和 tool roots。
- 扩展设置页：kernels、providers、mounted Apps、语音输入、远程 Matrix/Tuwunel 消息、代理、外观、诊断，以及 GitHub Copilot CLI 终端登录。
- 重做 Rooms 和 Contacts：员工/联系人变成显式 contact entities；本地 Room Ledger 持有主状态，Matrix/Tuwunel 作为可选投影写入通用 `remote` provenance。
- 打磨 shell UI：侧栏溢出、设置界面、联系人、room/member 管理、已加载 App 聊天输入框外壳，以及开发者标注工具栏清理。
- 刷新公开文档，包括 App spec、core decisions、technical reference、design guide、release process 和中英文文档。

## 配置说明

- 升级后请重启 OpenGrove local bridge，确保运行中的进程加载 `/developer/sessions`、App routes、extension routes 和 voice routes 等新路由。
- Mounted Apps 应在 `opengrove.app.json` 里声明包边界；`ui.kind` 可以指向文件工作台或 Web UI 预览。
- Matrix/Tuwunel 仍然是可选能力。本地 Rooms 持有主要状态；远程 room id 和同步元数据写入通用 `remote` provenance。
- 普通聊天输入框仍保留语音输入；开发者标注工具栏里未形成闭环的语音动作已移除。

## 验证

发布前已通过：

- `npm run check`
- `npm run test:harness`
- `npm run build:web`
- `npm pack --dry-run`
