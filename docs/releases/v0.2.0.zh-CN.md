# OpenGrove v0.2.0

这个版本收紧了 OpenGrove 的核心架构：Rooms 以 server-backed Room Ledger 为中心，同时 kernel integrations 从脆弱的命令行 fallback 转向更贴近协议原生的连接方式。

## 亮点

- Server-backed Rooms 和 Room Ledger 现在是 room state、messages、member targets、run ids、Matrix ids 和近期 room events 的事实来源。
- Matrix/Tuwunel 被定位为后端远程传输层。浏览器 UI 只读写本地 Rooms API state，不拥有 Matrix sync。
- Kernel integrations 现在优先使用更深的原生路径：
  - Codex：app-server JSON-RPC。
  - Claude Code：Claude Agent SDK。
  - Hermes：ACP over stdio，可选 HTTP gateway。
  - Pi：通过 `NativePiSession` 走 SDK in-process。
  - OpenClaw：Gateway WebSocket。
  - OpenCode、Copilot、Kimi、Kiro、DeepSeek：ACP over stdio。
  - Gemini、Qwen、Cursor：structured stream JSON CLI。
- OpenClaw 现在使用 Gateway WebSocket config，而不是 OpenAI-compatible HTTP config。
- README 和技术文档已刷新，包括更干净的品牌 lockup 和 `docs/TECHNICAL_REFERENCE.md`。

## 配置说明

OpenClaw 现在需要 Gateway endpoint：

```bash
OPENGROVE_KERNEL=openclaw
OPENGROVE_OPENCLAW_GATEWAY_URL=ws://127.0.0.1:PORT
OPENGROVE_OPENCLAW_GATEWAY_TOKEN=replace-with-gateway-token
```

Remote rooms 仍然需要后端 Matrix/Tuwunel 设置：

```bash
OPENGROVE_MATRIX_ENABLED=1
OPENGROVE_MATRIX_HOMESERVER_URL=https://matrix.example.com
OPENGROVE_MATRIX_USER_ID=@alice:matrix.example.com
OPENGROVE_MATRIX_ACCESS_TOKEN=replace-with-local-matrix-token
OPENGROVE_INVITE_BASE_URL=https://invite.example.com
```
