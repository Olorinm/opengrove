# OpenGrove v0.2.0

This release tightens OpenGrove's core architecture around a server-backed Room Ledger and moves kernel integrations toward protocol-native connections instead of brittle command-line fallbacks.

## Highlights

- Server-backed Rooms and Room Ledger are now the source of truth for room state, messages, member targets, run ids, Matrix ids, and recent room events.
- Matrix/Tuwunel is now treated as the backend remote transport layer. The browser UI reads and writes local Rooms API state; it does not own Matrix sync.
- Kernel integrations now prefer deeper native routes:
  - Codex: app-server JSON-RPC.
  - Claude Code: Claude Agent SDK.
  - Hermes: ACP over stdio, with optional HTTP gateway.
  - Pi: SDK in-process via `NativePiSession`.
  - OpenClaw: Gateway WebSocket.
  - OpenCode, Copilot, Kimi, Kiro, DeepSeek: ACP over stdio.
  - Gemini, Qwen, Cursor: structured stream JSON CLI.
- OpenClaw now uses Gateway WebSocket config instead of OpenAI-compatible HTTP config.
- README and technical docs were refreshed, including a cleaner brand lockup and `docs/TECHNICAL_REFERENCE.md`.

## Configuration Notes

OpenClaw now expects a Gateway endpoint:

```bash
OPENGROVE_KERNEL=openclaw
OPENGROVE_OPENCLAW_GATEWAY_URL=ws://127.0.0.1:PORT
OPENGROVE_OPENCLAW_GATEWAY_TOKEN=replace-with-gateway-token
```

Remote rooms still require backend Matrix/Tuwunel settings:

```bash
OPENGROVE_MATRIX_ENABLED=1
OPENGROVE_MATRIX_HOMESERVER_URL=https://matrix.example.com
OPENGROVE_MATRIX_USER_ID=@alice:matrix.example.com
OPENGROVE_MATRIX_ACCESS_TOKEN=replace-with-local-matrix-token
OPENGROVE_INVITE_BASE_URL=https://invite.example.com
```
