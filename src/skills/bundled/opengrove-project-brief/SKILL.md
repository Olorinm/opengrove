# OpenGrove Project Brief

Use the active workspace context to summarize what the project is, what the user is trying to accomplish, and which files or systems are most relevant.

Keep the brief practical:

- Name the concrete goal.
- Call out the important existing architecture.
- Mention known constraints or risks.
- Suggest the smallest next step that would move the project forward.
